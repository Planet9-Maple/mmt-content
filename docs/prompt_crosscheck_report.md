### 콘텐츠 형식 (실제 DB 기준)
```
1️⃣ [영어 문장 1]

2️⃣ [영어 문장 2]

3️⃣ [영어 문장 3]

[한국어 문장 1]
[한국어 문장 2]
[한국어 문장 3] [이모지]

⭐ {아이이름}:
[아이 영어 반응 1]
[아이 한국어 반응 1]

[아이 영어 반응 2]  ← 레벨 2-3에서 두 번째 반응이 있는 경우도 있음
[아이 한국어 반응 2] [이모지]
```

### 발견: 프롬프트에서 반영 안 된 것들
- **아이 반응이 2세트인 경우가 많음** (영어+한글 × 2개). 프롬프트에는 "1문장"으로 되어 있었음
- **{아이이름} 플레이스홀더** 사용. 프롬프트에 반영 필요
- **일요일은 복습일** — 새 콘텐츠 없음. 파이프라인에서 제외 필요
- **카테고리 컬럼 없음** — 주제명에서 추론해야 함

## 2. 규칙 vs 실제 DB — 불일치 발견

### 🔴 Level 1: 단어 수 규칙 위반 다수

**지침**: 3-6단어의 짧고 직관적 문장
**실제 DB**: 8단어 이상 문장이 빈번

| 실제 문장 | 단어 수 | 판정 |
|-----------|---------|------|
| "You want it again? Okay, one more time." | 8 | ⚠️ 초과 |
| "How is it? Nice and warm! Bubbles today?" | 8 | ⚠️ 초과 |
| "Time to wash your hair. Close your eyes." | 8 | ⚠️ 초과 |
| "All done! Your hair is so fluffy." | 7 | ⚠️ 초과 |
| "Wow, so tall! What should we make now?" | 8 | ⚠️ 초과 |

**분석**: 실제 제작 과정에서 "3-6단어"는 엄격한 제한이 아니라 "느낌"으로 적용된 것으로 보임. 2개의 짧은 문장을 한 줄에 합치는 경우(예: "Nice and warm! Bubbles today?") 총 단어 수는 늘어나지만 각각은 짧음.

**→ 프롬프트 수정 제안**: "3-6단어의 짧은 절 중심. 한 줄에 짧은 절 2개를 조합하는 것은 허용 (예: 'Nice and warm! Bubbles today?'). 단, 한 절이 6단어를 넘지 않아야 함."

---

### 🔴 Level 2 아이 반응: 규칙 위반 다수

**지침**: 2-3단어, chunk 표현. 조동사/to부정사 금지 (레벨3 표현)
**실제 DB**: 조동사/to부정사가 포함된 4-5단어 반응이 다수

| 실제 반응 | 단어 수 | 위반 사항 |
|-----------|---------|-----------|
| "Can I put toys in?" | 5 | can + 동사원형 (레벨3 구조) |
| "I want to try!" | 4 | want to (레벨3 구조) |
| "Want to go!" | 3 | want to (레벨3 구조) |
| "I want to!" | 3 | want to (레벨3 구조) |
| "I'll try." | 2 | 조동사 will (레벨3 구조) |
| "Can I touch?" | 3 | can + 동사원형 (레벨3 구조) |
| "I can do it!" | 4 | can + 동사원형 (레벨3 구조) |

총 8건 이상의 레벨 규칙 위반.

**분석**: 실제 제작 중 "자연스러움"을 우선시하면서 레벨 경계가 유연하게 적용된 것으로 보임.

**→ 프롬프트 수정 제안 (2가지 옵션)**:
- **옵션 A (규칙 강화)**: 기존 지침 그대로 유지. Step 4 검수에서 이런 표현을 잡아내어 감점.
- **옵션 B (현실 반영)**: 레벨 2 아이 반응 규칙을 "기본적으로 2-3단어 chunk. 단, 'I can do it!', 'I want to try!' 같은 매우 기본적인 패턴은 조건부 허용" 으로 완화.

**Jun에게 결정 필요**: 기존 규칙을 엄격히 적용할지, 실제 DB 패턴을 인정할지?

---

### 🟡 아이 반응 개수: 1개가 아니라 2개

**지침**: "⭐ 아이 대답(1문장 영어+한글)"
**실제 DB**: 대부분 2세트 (영어+한글 반응이 2쌍)

```
⭐ {아이이름}:
Good morning, Mommy!     ← 반응 1
엄마 굿모닝!

Let's go eat!            ← 반응 2
아침 먹으러 가자! 😆
```

**→ 프롬프트 수정**: "아이 반응 2세트 생성. 각 세트는 영어 1문장 + 한국어 1문장. 첫 번째 반응은 엄마의 말에 대한 직접적 반응, 두 번째는 상황에 대한 자발적 발화."

---

### 🟡 카테고리 분류 부재

DB에 category 컬럼이 없어서, 파이프라인이 중복 체크를 하려면 주제명에서 카테고리를 추론해야 함.

**주제를 분석해보면 대략적 카테고리**:
- **식사/간식**: 수저 놓기, 앉아서 먹자, 세 입만 더, 과일 고르기, 반찬, 더 줄까
- **위생/몸 관리**: 손 씻기, 목욕, 머리 말리기, 손톱 깎기, 양치, 로션
- **놀이**: 블록놀이, 숨바꼭질, 그림 그리기, 퍼즐, 풍선, 공놀이
- **외출/이동**: 10분 뒤 나가기, 차 조심, 신발 신기
- **정리/규칙**: 장난감 정리, 빨래통, 소리치지 않기
- **취침**: 잘 시간, 자기 전 책, 사랑해 잘자
- **계절/특별**: 크리스마스, 새해, 새싹, 벚꽃

**→ 파이프라인 수정**: Step 0 또는 Step 1에서 주제를 카테고리로 자동 분류하는 로직 추가. 또는 DB에 category 컬럼을 수동으로 한 번 태깅 (130개니까 30분이면 가능).

## 3. 프롬프트 수정이 필요한 항목 종합

### Step 3 (문장 생성) 수정 사항

| # | 현재 프롬프트 | 수정 필요 | 이유 |
|---|-------------|-----------|------|
| 1 | "레벨1: 3-6단어" | "3-6단어의 짧은 절. 한 줄에 짧은 절 2개 조합 허용" | 실제 DB에서 짧은 절 2개를 한 문장으로 쓰는 패턴이 표준 |
| 2 | "아이 반응 1문장" | "아이 반응 2세트 (영어+한글 × 2)" | 실제 DB 형식 |
| 3 | 출력 형식에 {아이이름} 없음 | `⭐ {아이이름}:` 플레이스홀더 포함 | 실제 서비스에서 아이 이름 치환 |
| 4 | 레벨2 아이 반응 "2-3단어 chunk만" | Jun 결정 필요 (강화 vs 완화) | DB에 위반 다수 |
| 5 | 출력이 JSON | JSON + 실제 발송 형식 둘 다 생성 | 어드민 입력 시 바로 붙여넣기 위해 |

### Step 4 (검수) 수정 사항

| # | 현재 프롬프트 | 수정 필요 | 이유 |
|---|-------------|-----------|------|
| 1 | "중복 체크" 기준 모호 | 구체적 기준: "같은 카테고리 내 2개월 이내 동일 행동 소재이면 중복" | 카테고리 분류 기반 |
| 2 | 아이 반응 검수 기준 | 레벨2 "2세트 반응 각각이 2-3단어 chunk인지" 확인 | 반응 2개 형식 반영 |

| # | 항목 | 내용 |

## 4. 실제 DB가 보여주는 "좋은 콘텐츠" 패턴

DB를 분석하면서 발견한, 프롬프트에 반영하면 좋을 패턴들:

### 레벨 간 확장 패턴 (잘 된 사례)
```
주제: 🕙 10분 뒤에 나가요

Level 1: "We're leaving in ten minutes."          (6w, 단일 정보)
Level 2: "We're leaving in ten minutes, okay?"     (7w, +확인 요청)
Level 3: "We're heading out in ten minutes,        (12w, +이유+행동유도)
          so let's start getting ready."
```
→ 핵심 구조는 유지하면서 정보량만 늘어남. 이 패턴을 프롬프트에 명시적으로 예시로 넣어야 함.

### 아이 반응 레벨 간 확장 패턴
```
주제: 📚 같이 책을 읽어요

Level 2: "I want this one!" (4w)     → 단순 의사표현
Level 3: "I want to read this one!" (6w) → want to + 동사 구조 확장
```
→ 같은 의미의 반응이 레벨에 따라 구조적으로 확장되는 패턴. 좋은 예시.

### 한국어 톤 패턴
```
Level 1 한국어: "아침 먹으러 가자 🥰"           → 짧고 직접적
Level 2 한국어: "자, 아침 먹으러 가자 🥰"        → "자," 추가로 부드러움
Level 3 한국어: "이제 부엌으로 가서 아침 먹자 🥰"  → 구체적 장소 + 설명
```
→ 한국어도 레벨에 따라 확장됨. 프롬프트에서 이 패턴을 명시해야 함.
